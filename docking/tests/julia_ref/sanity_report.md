# スコア妥当性チェックレポート (A-3)

`docking_canonical_overrides_buggy.jl`（ノートブック raw）と `docking_canonical.jl`（B1/B3 修正込み）で、1KXQ の top-10 Decoy ポーズに対する `docking_score_elec(α=0.01, β=3.0)` を比較した結果。

## サマリ

| | min | max | mean | std |
|---|---|---|---|---|
| BUGGY (ノートブック原典) | -4794 | -2949 | -3538 | 786 |
| FIXED (B1+B3 修正後) | -3457 | -1672 | -2256 | 728 |

## 1. B1 修正による NaN 発生と対処

ナイーブに `grid_imag = grid_charge ./ grid_dis` → `grid_imag .= grid_charge ./ grid_dis` へ直しただけでは、`grid_dis` がゼロの格子セル（rcut 8Å 内に原子無し）で **NaN/Inf** を生む。これを

```julia
grid_imag .= ifelse.(grid_dis .> zero(T),
                     grid_charge ./ max.(grid_dis, eps(T)),
                     zero(T))
```

で安全化した。元のバグ版は「そもそも `grid_imag` を更新しない」ので NaN 問題は表面化していなかった。

## 3. 観察と判断

- **B1 修正は正しい方向性**：元コードでは ELEC 寄与がまともに反映されず、β パラメータの学習がほぼ効いていなかった。修正で ELEC 項が意味のある負・正の値を取り始め、α/β/iface/charge の勾配バランスが正常化すると期待できる。
- **score_total の絶対値がシフト**：α, β, iface, charge 全てに対する損失勾配が変わるので、修論の数値結果（Rank 42→10 等）は修正後には厳密には再現しない可能性が高い。ただし Hit/Rank が改善する**方向性**は保存されるはず。
- **B4（Σq/Σr vs Σq/r）は未着手**：現状は修論通り `Σq/Σr`（B1 修正済）。物理的に正しい `Σq/r` に倒すかは別途判断。判定タイミングは A-4（勾配整合確認）後が良い。
